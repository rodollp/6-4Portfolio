# 구현 기능

## 플레이 방법
- wasd 방향키
- 마우스 좌클릭 = 공격
- 마우스 커서 = 정면
- n키 => 스테이지 클리어 후 다음 스테이지 이동
- h키 => 포션 아이템 사용
- p키 => 무기 아이템 사용
- g키 => 문과의 상호작용, 문 앞에서 입력시 문 오브젝트 2초간 소멸상태

## 플레이어

- `Unity Input System`을 사용하여 플레이어 이동 구현
- `Rigidbody` 기반 이동 및 점프 시스템 구현
- `Raycast`를 사용한 바닥 감지 처리
- 레벨이 오를 때 마다 공격력,최대 체력,체력 상승

## 몬스터 AI

- 플레이어 감지 범위 및 공격 범위 구현
 - `Dot`(내적)과 인지 범위를 통해 시야 범위 구현
 - `sqrMagnitude`를 사용한 거리 계산 최적화
 - `Quaternion.Slerp`(구면 선형 보간)을 이용하여 매끄러운 회전 구현
- FSM(Finite State Machine) 기반 몬스터 상태 관리
  - Idle
  - Chase
  - Attack

## 전투 시스템

- 범위 공격 시 공격 범위 내 몬스터 넉백 구현
- `SphereCast`를 이용하여 구 안의 있는 적을 공격
- 몬스터 공격에 공격 딜레이를 줘서 1초마다 범위에 있는 플레이어 자동공격 구현

## 스테이지 시스템

- List 자료구조를 사용하여 현재 스테이지 몬스터 관리, 현재 Sector에 있는 Room 관리
- `StageManager`에서 `List<Monster>`를 통해 살아있는 몬스터 추적
- 몬스터 사망 시 이벤트 기반으로 스테이지 진행 처리

## 아이템 드랍 시스템

- 몬스터별 드랍 아이템 배열을 구성하여 랜덤 드랍 구현
- `Random.Range`를 이용해 드랍 아이템 중 하나를 선택
- 드랍 아이템 생성 시 위치 보정을 통해 바닥에 박히지 않도록 처리
- 드랍 아이템에 회전 및 상하 이동 연출 적용

## 데이터 관리

- `DropData`,`ItemData`, `StageData`를 사용하여 데이터와 로직 분리
- 아이템 이름, 타입, 수치 데이터를 코드 수정 없이 Inspector에서 관리
- 스테이지별 몬스터 종류, 스폰 수, 보스 스테이지 여부를 데이터로 관리

## 인벤토리
- List 자료구조를 사용하여 인벤토리 구성
- `InputSystem`과 `sqrMagnitude`를 이용해 가장 가까운 거리에 있는 아이템 픽업 기능 추가
- 포션과 무기로 종류 세분화,회복 및 공격력 상승 효과
- 몬스터 사망 시 드랍 테이블 기반으로 아이템 오브젝트 생성
- 플레이어가 가까운 아이템을 탐색하여 픽업하면 인벤토리에 추가

## 배틀 시스템

- 몬스터 사망 이벤트를 구독하여 보상 지급 및 아이템 드랍 처리
- 플레이어 경험치 및 재화 획득 처리
- 전투 결과에 따라 스테이지 진행 로직과 연동

## Collider 적용
- 방 안에 동,서,남,북으로 있는 문에 `BoxCollider`에 `IsTrigger`를 체크하여 키 입력시 문 제거
- `IsTrigger`체크시 오브젝트 통과를 막기위해서 물체와 충돌 범위는 분리하여 적용

## 이벤트 구조
- 몬스터 사망 시 `OnDead` 이벤트를 호출하여 보상, 아이템 드랍, 스테이지 진행을 분리
- `BattleManager`, `StageManager`가 몬스터 사망 이벤트를 구독하는 구조로 구현

## 길찾기 시스템

- 플레이어와 가장 가까운 몬스터가 있는 방 탐색 기능 구현
- RoomNode 기반 A* 경로 탐색 구현
- Gizmo를 활용한 경로 시각화
